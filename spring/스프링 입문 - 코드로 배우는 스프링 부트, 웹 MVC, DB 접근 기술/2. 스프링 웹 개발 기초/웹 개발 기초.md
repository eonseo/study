# 1. 정적 컨텐츠

- 웹 애플리케이션이 정적 자원을 제공하는 방식
- HTML, CSS, JavaScript, 이미지 같은 파일을 서버에서 그대로 내려주는 기능

```plain text
By default, Spring Boot serves static content from a directory called /static(or /public or /resources or /META-INF/resources) in the classpath or from the root of the ServletContext.
```

## 기본 제공 위치

- /static
- /public
- /resources
- /META-INF/resources

이 경로의 파일은 별도의 컨트롤러 없이도 HTTP 요청으로 접근 가능

# 2. MVC 와 템플릿 엔진

## Model

- 데이터/비즈니스 로직

## View

- 화면을 그리는데 집중

## View Resolver

- Spring MVC 에서 컨트롤러가 반환한 뷰 이름을 실제 뷰 객체로 매핑해주는 역할을 하는 컴포넌트

```java
@GetMapping("hello-mvc")
    public String helloMvc(@RequestParam("name") String name, Model model) {
        // http://localhost:8080/hello-mvc?name=spring
        // -> 모델이 "name"="spring" 저장
        model.addAttribute("name", name);
        return "hello-template";
    }
```

## `뷰 이름`

- 컨트롤러가 반환하는 문자열 (`hello-template`)
- 단순히 문자열로, 어떤 화면을 보여줄지를 나타내는 식별자 역할
- view resolver 가 이 이름을 해석해서 실제 뷰 파일을 찾아냄

👉 여기서는 `hello-template` -> `resources/templates/hello-template.html` 로 매핑

## 뷰 객체

- view resolver 가 뷰 이름(`hello-template`)을 실제 템플릿 파일과 연결해 만든 렌더링 객체
- 뷰 객체는 Spring MVC 내부에서 화면 렌더링을 담당하는 객체로, Model 데이터를 받아 HTML 을 생성
- 즉, `hello-template` 이라는 이름이 ThymeleafView 객체로 바뀌어 실행

## 요청 흐름 예시

1. 사용자가 `http://localhost:8080/hello-mvc?name=spring` 요청

2. 컨트롤러 실행 → `model.addAttribute("name", "spring")` 저장

3. 컨트롤러 반환값 `hello-template` → 뷰 이름

4. View Resolver가 `hello-template`을 해석 → 뷰 객체(ThymeleafView) 생성

5. 뷰 객체가 hello-template.html을 렌더링하면서 Model의 "name"="spring"을 치환

6. 최종 HTML 응답:

```html
<p>hello spring</p>
```
