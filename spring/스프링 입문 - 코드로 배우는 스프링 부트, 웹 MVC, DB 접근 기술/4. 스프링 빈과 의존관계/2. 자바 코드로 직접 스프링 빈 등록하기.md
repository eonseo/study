- service, repository 단에서 @Service, @Repository 애너테이션을 삭제해보면 아래와 같이 어플리케이션 실행에 실패한다.

```bash
UnsatisfiedDependencyException:
No qualifying bean of type '...MemberRepository' available
```

- MemberService 생성자가 MemberRepository 를 필요로 하는데
- 스프링이 MemberRepository 를 스프링 빈으로 등록하지 않아서 찾을 수 없다는 뜻

👉 SpringConfig 파일을 만들어서 명시적으로 `@Configuration`, `@Bean` 을 등록해준다.

**참고**

- DI 에는 필드 주입, setter 주입, 생성자 주입 3가지 방법이 있다.

**주의**

- `@Autowired` 를 통한 DI 는 helloController, MemberService 등과 같이 스프링이 관리하는 객체에서만 동작함
- 스프링 빈으로 등록하지 않고 내가 직접 생성한 객체에서는 동작하지 X
